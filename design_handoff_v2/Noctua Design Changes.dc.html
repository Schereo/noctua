4C Discipline — why she stays charming
  • One living owl per screen. The rail owl is the only one that moves; every owl on a sheet is a static stamp.
  • Blink is rare: 120ms every 8–14s (randomized per mount via animation-delay), never while listening, never two owls in sync.
  • Motion budget ≤ 1.1px (scan) — no bouncing, no rotation, no scale. prefers-reduced-motion: all poses freeze to their static frame.
  • Never near reading or writing: no owl in mail bodies, list rows, the composer, or print/PDF output. Never in the masthead — the Noctua wordmark is the brand there.
  • Ink only. Accent is reserved for the two-dot toast form on ink ground.
  • Poses mean something. Asleep = can't work (no key) or nothing to do (zero states). Scan = working. Listen = mic open. Never decorative mood-switching.

  • Typing streams live hits from the local index (FTS + embeddings, the existing useSemanticSearch) — instant, free, debounced 300ms. No LLM call yet.
  • ↑↓ moves through hits; ↵ on a hit opens the thread (existing palette jump logic). ↵ with none selected asks the owl — the ask row is always first and shows exactly what will be spent.
  • Asking converts the query into a conversation (the 1a layout: question title, streamed answer, sources = the hits). The input stays at the sheet top for follow-ups; n starts fresh.
  • Entry points: / and ⌘F open this view with the input focused (both currently open the palette); ⌘5 / masthead OWL navigates without focusing.

WHY THIS SPLIT

"Find that mail" and "answer this question" share one retrieval engine but differ in cost and intent. Fusing them into one input with an explicit ask row keeps lookup instant and makes the token-spending step a visible, deliberate escalation — while ⌘K goes back to being a command palette that opens in 0ms and never surprises you with a search.

QUEUE RULES
  • One toast visible at a time, bottom-center 26px, max-width 70% — the existing PaperToast geometry.
  • Priority: countdown > error > action > info. A countdown preempts an info toast (queued info shows after, if still fresh < 8s).
  • Action toasts live exactly as long as their offer (filed = the 5s archive window). Info keeps 3.6s. Errors persist.
  • Undo-send becomes the countdown variant: rec-dot pulses, hairline progress runs down, UNDO ⌘Z cancels — replacing OutboxToast wholesale.
  • Update notice becomes an info toast with a RESTART action instead of the top banner.
  • Toasts announce via role="status" (errors: role="alert"); actions are real buttons.
